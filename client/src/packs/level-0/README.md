# DARE OS Pack – Nível 0: Artesanal

> **Tagline:** Nenhuma IA. Só cérebro, mão e suor.

## Visão geral

Nível 0 é para projetos em que **nenhuma IA** será usada. Tudo é feito à moda antiga: o designer conduz pesquisa, criação e entrega com ferramentas tradicionais.

## Como usar em 30 minutos

1. **Defina o escopo**: Documente explicitamente por que IA não será usada neste projeto.
2. **Estabeleça critérios**: Crie critérios de aceite manuais e checkpoints de revisão.
3. **Preencha o governance**: Use `governance.md` para registrar a decisão de Nível 0.
4. **Mantenha consistência**: Todos os artefatos são produzidos sem assistência de IA.

## Quando usar

- Projetos sensíveis à identidade
- Branding rígido com requisitos não verbalizáveis
- Entregas finais onde controle total é essencial
- Ambientes regulatórios que proíbem IA
- Quando o impacto de falha é alto e regras são indefinidas

## Quando NÃO usar

- Quando há tarefas repetitivas que a IA faria bem
- Quando tempo é crítico e IA poderia acelerar
- Quando regras são claras e validação é possível
- Quando research e síntese consumiriam tempo desproporcional

## Artefatos do Pack

| Arquivo | Descrição |
|---------|-----------|
| `PRD.md` | Product Requirements (lite) — requisitos e entregas |
| `SDD.md` | Specification/Design Doc (lite) — processo e ferramentas |
| `governance.md` | Decisão de Nível 0, justificativa, responsáveis |
| `eval/rubric.md` | Rubrica de avaliação manual |
| `prompts/registry.yaml` | N/A — placeholder (Nível 0 não usa prompts) |

## Gates obrigatórios

- [ ] Decisão explícita de não usar IA documentada em `governance.md`
- [ ] Critérios de aceite definidos manualmente
- [ ] Revisão humana em todos os checkpoints
- [ ] Justificativa clara para Nível 0

## Checklist de início

- [ ] Escopo definido sem dependência de IA
- [ ] Critérios de aceite documentados
- [ ] Responsáveis e revisor definidos
- [ ] Governance preenchido e assinado
